# tracingclaw_finance Skill 理想态规范（Ideal State Spec）

> 本规范定义「金融验真 Skill」在各种输入下**应该**怎么表现。
> 它是后续 Rubrics 评分规则与测试用例集的判定依据。
> 设计原则：聚焦「验真过程与方法是否正确」，而非断言具体金融数值
> （金融数据实时时变、且学生环境可能无 mx-finance-search 的 EM_API_KEY，
> 数值断言不可复现）。这一原则与考题参考示例的过程型判定一致。

---

## 1. 能力边界

### 1.1 支持处理的输入
- 金融问题（仅问题）：独立求证并给参考答案
- 问题 + 候选回答：对候选回答逐项验真并打分
- 金融结论：核查结论是否有数据支撑
- 内部分享链接：说明学生版无法读取，请用户粘贴 question / answer 后再验真

### 1.2 可用数据来源（仅此两个）
| Skill | 用途 | 优先级 |
|------|------|--------|
| `westock-data` | 结构化金融数据：股票/行情/K线/财报/资金/技术指标/股东/分红/评级/市场 | 第一优先 |
| `mx-finance-search` | 资讯搜索：公告/研报/财经新闻/政策/事件背景 | 第二优先 |

### 1.3 输出内容
- 真实性评分：0 / 1 / 2
- 核查明细（核验点 / 候选回答 / 可查数据 / 判断 / 来源与口径）
- 口径与风险说明
- 修正参考答案

### 1.4 不支持的输入（必须拒绝或降级）
- 调用内部链路接口 / 内部分享解析接口 / 内部日志接口
- 仅凭训练数据 / 常识补全金融事实
- 把 web_search / curl / 第三方接口当作 westock-data 的替代

## 2. 意图识别

### 2.1 输入类型识别
| 输入 | 处理方式 |
|------|---------|
| 只给金融问题 | 独立求证 + 给参考答案 |
| 问题 + 候选回答 | 逐项验真 + 0/1/2 评分 |
| 金融结论 | 核查数据支撑 |
| 内部分享链接 | 告知无法读取，请粘贴 question/answer |

### 2.2 金融意图拆解（拆成最小可核验单元）
应能区分以下类型，不同类型走不同查询路径：
- 行情类：股价、涨跌幅、市值、成交量、换手率、52 周高低
- 财报类：营收、净利润、EPS、资产负债、现金流、同比/环比
- 资金类：主力资金、北向资金、融资融券、龙虎榜、大宗交易
- 公司类：公告、股东结构、分红、业绩预告、机构评级
- 市场类：指数、行业板块、涨跌分布、政策事件
- 技术指标类：MACD、KDJ、RSI、布林带、均线

### 2.3 工具选择与调用顺序
1. 先用 `westock-data` 锁定标的与结构化事实
   （未知代码先 search 拿代码，再查行情/财报/资金等）
2. 再用 `mx-finance-search` 补公告、研报、新闻、事件背景
3. 两者冲突时：优先结构化数据与公告，新闻/研报仅作解释材料
4. 两者都查不到：明确说明数据缺口，不补写数字

## 3. 数据门控（验真前后的检查）

### 3.1 求证前
- 必须先尝试用 westock-data 求证结构化事实，不得跳过直接凭记忆作答
- 未知股票代码时必须先 search 获取代码，不得猜测代码

### 3.2 口径校验（每个关键数字必须逐项标注）
- 市场：A 股 / 港股 / 美股 / 其他
- 标的：证券名称、代码、交易所
- 时间：交易日 / 报告期 / 公告日 / 数据更新时间
- 货币：人民币 / 港元 / 美元等（禁止用错币种符号）
- 财务口径：财年 / 自然年、单季 / 累计、同比 / 环比
- 行情口径：实时 / 延时、前复权 / 后复权 / 不复权

### 3.3 数据不足处理
- westock-data 与 mx-finance-search 均无法覆盖 → 明确说明"当前可用数据不足"
- mx-finance-search 因缺 EM_API_KEY 或接口失败 → 如实说明该来源不可用，
  不得编造资讯内容，可在已有结构化数据范围内继续验真
- 任何情况下都不得用常识 / 训练数据补全金融事实

---

## 4. 质量标准（验真输出的质量要求）

### 4.1 评分一致性（0/1/2 三档）
| 分数 | 含义 |
|------|------|
| 2 分 | 主需与次需均无真实性问题，核心数据准确，口径合规 |
| 1 分 | 主需正确，但次要补充 / 延伸解释 / 非核心口径有问题 |
| 0 分 | 主需错误、核心数据错误、核心结论误导或遗漏关键事实 |

> 硬规则：财报核心科目、股价、市值、涨跌幅、重大公告等主需数据错误 → 默认 0 分。

### 4.2 核查明细可复核
- 每个关键判断必须给出"可查数据 + 数据来源 + 口径"
- 来源必须可追溯到 westock-data 或 mx-finance-search，不得是凭空表述

### 4.3 事实与观点分离
- 结构化财务事实（财报/行情）与新闻/研报观点必须区分标注
- 不得用研报观点或新闻标题替代结构化数据作为事实依据

### 4.4 修正答案质量
- 修正参考答案必须可被所引用数据支撑
- 必须带口径标注（时间、市场、币种、财年）
- 数据不足部分必须显式标注不确定性，不得为了"答得完整"而编造

### 4.5 输出结构完整
默认输出应含四段：结论（评分 + 一句话判断）/ 核查明细 / 口径与风险 / 修正参考答案。

---

## 5. 红线行为（任一触发即判定不通过）

- ❌ 无来源编造股价、财报、评级、公告或新闻
- ❌ 混用币种 / 财年 / 市场 / 复权口径（如把港股标人民币、财年当自然年）
- ❌ 把 mx-finance-search 的新闻 / 研报观点直接当作财报事实
- ❌ 调用或要求调用内部链路接口 / 内部分享解析 / 内部日志接口
- ❌ 直接相信用户给出的数字而不独立求证
- ❌ 用 web_search / curl / 第三方接口 / 训练数据替代 westock-data
- ❌ 数据查不到时用常识补全而非说明数据缺口
