# tracingclaw_finance Skill 评分规则（Rubrics）

> 本规则将「tracingclaw 理想态规范」翻译为可打分的量规，供人工或基座 agent 自动判分使用。
> 框架与 db_report 一致：先红线检查（重罚制）→ 再五维度加权打分 → 映射等级。
> 设计原则：评分聚焦验真过程与方法的正确性，不依赖具体金融数值（数值实时时变、
> 且学生环境可能无 mx-finance-search 的 EM_API_KEY）。

---

## 一、评分流程总览

```
输入用例输出
   ↓
① 红线检查（重罚制）
   ├─ 触发任一红线 → 总分压入「红线区间 0-3 分」，按严重程度给分
   └─ 未触发 → 进入②
   ↓
② 五维度加权打分（满分 100）
   ↓
③ 映射等级
```

---

## 二、红线检查（重罚制，不直接否决）

触发任一红线，总分被压入 **红线区间（0-3 分 / 满分 100）**，
由评审根据「违规严重程度」与「是否有意为之」在 0-3 分内给出最终分。

### 红线项

| 编号 | 红线项 |
|------|--------|
| R1 | 无来源编造股价 / 财报 / 评级 / 公告 / 新闻 |
| R2 | 混用币种 / 财年 / 市场 / 复权口径（如港股标人民币、财年当自然年）|
| R3 | 把 mx-finance-search 的新闻 / 研报观点直接当作财报事实 |
| R4 | 调用或要求调用内部链路接口 / 内部分享解析 / 内部日志接口 |
| R5 | 直接相信用户数字不独立求证，或用 web_search/curl/训练数据替代 westock-data |

### 红线区间给分标准

| 残留分 | 适用情况 |
|--------|---------|
| 3 | 轻微触线且有补救迹象（如已标注不确定性） |
| 2 | 触线但无恶意、影响有限 |
| 1 | 明确触线、影响较大 |
| 0 | 严重违规、输出完全不可信 |

> 红线一旦触发，无论其他维度做得多好，总分上限即为 3 分。

## 三、五维度加权打分（满分 100）

| 维度 | 权重 | 评什么 | 满分条件 |
|------|------|--------|---------|
| **D1 求证过程** | 25 | 先查 westock 再答、未知代码先 search、工具调用顺序正确 | 优先结构化求证，顺序正确，不凭记忆 |
| **D2 口径一致性** | 25 | 市场/币种/时间/财年/复权口径逐项标注且正确 | 关键数字均带正确口径标注 |
| **D3 评分合理性** | 20 | 0/1/2 分符合评分规则、主需错误判 0 | 评分档位与理由匹配规则 |
| **D4 输出可复核性** | 20 | 来源可追溯、事实/观点分离、修正答案有支撑 | 每判断有来源；事实观点分开；修正答案可被支撑 |
| **D5 降级处理** | 10 | 数据不足/工具失败时如实说明、不编造 | 正确说明缺口，不补写数字 |

**合计：25 + 25 + 20 + 20 + 10 = 100**

### 各维度扣分项明细

**D1 求证过程（25）**
- 未调 westock-data 直接凭记忆/常识作答：-15
- 未知股票代码未先 search 就猜代码：-5
- 工具调用顺序错误（先用新闻/研报当事实依据）：-5

**D2 口径一致性（25）**
- 币种标错（如港股标人民币、美股标人民币）：-15
- 财年/自然年、单季/累计、同比/环比混用：-10
- 缺失市场/时间/复权等关键口径标注：每类 -5

**D3 评分合理性（20）**
- 主需数据错误却未判 0 分：-15
- 0/1/2 档位与给出理由不匹配：-10
- 未输出 0/1/2 评分：-10

**D4 输出可复核性（20）**
- 关键判断无数据来源：每处 -8
- 事实与观点未分离（新闻观点混入财报事实陈述）：-8
- 修正答案无数据支撑或无口径标注：-6

**D5 降级处理（10）**
- 数据查不到时编造而非说明缺口：-8
- mx-finance-search 不可用（缺 key/失败）时未如实说明：-5

> 各维度扣分以扣至该维度 0 分为下限，不出现负分。

## 四、等级映射

| 总分 | 等级 | 含义 |
|------|------|------|
| 90-100 | 优秀 | 完全符合理想态，可作标杆 |
| 75-89 | 合格 | 主体正确，少量非致命瑕疵 |
| 60-74 | 待改进 | 有明显问题但方向正确 |
| 4-59 | 不合格 | 关键维度失败 |
| 0-3 | 红线重罚 | 触发任一红线 |

---

## 五、打分输出格式建议

```
- redline_triggered: <true/false，及触发编号>
- redline_residual: <0-3，仅 redline_triggered=true 时填>
- D1_verification_process: <0-25>
- D2_caliber_consistency: <0-25>
- D3_scoring_validity: <0-20>
- D4_traceability: <0-20>
- D5_graceful_degradation: <0-10>
- total: <0-100>
- grade: <优秀/合格/待改进/不合格/红线重罚>
- deduction_notes: <逐条扣分原因>
```
