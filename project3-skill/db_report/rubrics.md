# db_report Skill 评分规则（Rubrics）

> 本规则将「db_report 理想态规范」翻译为可打分的量规，供人工或基座 agent 自动判分使用。
> 评分流程：先做红线检查（重罚制）→ 再做五维度加权打分 → 映射等级。

---

## 一、评分流程总览

```
输入用例输出
   ↓
① 红线检查（重罚制）
   ├─ 触发任一红线 → 总分压入「红线区间 0-3 分」，按严重程度给分，不再走加权
   └─ 未触发 → 进入②
   ↓
② 五维度加权打分（满分 100）
   ↓
③ 映射等级
```

---

## 二、红线检查（重罚制，不直接否决）

触发任一红线，该用例总分被压入 **红线区间（0-3 分 / 满分 100）**，
由评审根据「违规严重程度」与「是否有意为之」在 0-3 分内给出最终分。

### 红线项

| 编号 | 红线项 |
|------|--------|
| R1 | 编造 / 估算 / 硬编码 TPS / QPS / P95 / P99 / duration 等性能数字 |
| R2 | 缺数据时强行生成正式报告 |
| R3 | 尝试连接内部库 / HTTP API / 用 task_id 等标识查询补全数据 |
| R4 | 将 5 个 sysbench 场景合并为均值输出 |
| R5 | 用 pandoc 等工具批量转换替代三格式独立生成 |

### 红线区间给分标准

| 残留分 | 适用情况 |
|--------|---------|
| 3 | 轻微触线且有补救迹象（如局部推测已明确标 `[待确认]`） |
| 2 | 触线但无恶意、影响有限 |
| 1 | 明确触线、影响较大 |
| 0 | 严重违规、输出完全不可信 |

> 说明：红线一旦触发，无论其他维度做得多好，总分上限即为 3 分。

## 三、五维度加权打分（满分 100）

| 维度 | 权重 | 评什么 | 满分条件 |
|------|------|--------|---------|
| **D1 数据可信** | 30 | 数据真实可追溯、空值率达标、来源清晰 | 所有数字可追溯到输入；空值率 0%；source_info 完整 |
| **D2 意图识别** | 20 | report_type、场景中英映射、AND/OR 逻辑、数据源类型 | 四项全对 |
| **D3 报告质量** | 30 | 5 场景独立、洞察分级、图表数、结论数、必含元素 | 全部满足对应类型基线 |
| **D4 格式交付** | 12 | 三格式独立生成、文件大小达下限、docx/html 样式 | md/docx/html 齐全且达标 |
| **D5 失败处理** | 8 | 缺数据 / 异常时正确停止并给对错误码与提示 | 正确识别 keyword_only 或对应错误码并停止 |

**合计：30 + 20 + 30 + 12 + 8 = 100**

### 各维度扣分项明细

**D1 数据可信（30）**
- 数字无法溯源到输入文件：每处 -15
- 空值率 > 0% 且未显式处理：-10
- source_info 缺失 type/value/rows_fetched 任一：-5

**D2 意图识别（20）**
- report_type 识别错误：-15
- 场景中英映射错误或未映射：-5
- AND/OR 逻辑判断错误：-5
- 数据源类型识别错误：-5

**D3 报告质量（30）**
- 5 场景被合并 / 未独立分析：-10
- L3 推测越位（出现在第 1-4 章正文）：-5
- 图表数未达对应类型基线：-5
- 核心结论数未达对应类型基线：-3
- 类型必含元素缺失（如 comparison 缺公平性说明）：-5

**D4 格式交付（12）**
- 缺少 md/docx/html 任一格式：每种 -6
- 文件大小未达下限（md≥5KB / docx≥10KB / html≥8KB）：-3
- docx/html 样式不合规（字体 / 表头色 / 字体 fallback）：-3

**D5 失败处理（8）**
- 缺数据 / 异常时未停止流程：-5
- 错误码错误或缺少补充数据提示：-3

> 各维度扣分以扣至该维度 0 分为下限，不出现负分。

---

## 四、等级映射

| 总分 | 等级 | 含义 |
|------|------|------|
| 90-100 | 优秀 | 完全符合理想态，可作标杆 |
| 75-89 | 合格 | 主体正确，少量非致命瑕疵 |
| 60-74 | 待改进 | 有明显问题但方向正确 |
| 4-59 | 不合格 | 关键维度失败 |
| 0-3 | 红线重罚 | 触发任一红线 |

---

## 五、打分输出格式建议

每条用例评分时建议输出：

```
- redline_triggered: <true/false，及触发编号>
- redline_residual: <0-3，仅 redline_triggered=true 时填>
- D1_data_trust: <0-30>
- D2_intent: <0-20>
- D3_report_quality: <0-30>
- D4_format: <0-12>
- D5_failure_handling: <0-8>
- total: <0-100>
- grade: <优秀/合格/待改进/不合格/红线重罚>
- deduction_notes: <逐条扣分原因>
```
